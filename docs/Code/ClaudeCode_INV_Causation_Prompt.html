Context Prompt: Bull Trout MPVA — Invasive Species Causation Analysis

Project Overview

This is a multi-population viability analysis (MPVA) for bull trout (Salvelinus confluentus) across ~110 local populations in the northern Rocky Mountains (primarily Montana, Idaho, BC). The model is a state-space Ricker growth model implemented in TMB (Template Model Builder) with a three-level spatial hierarchy: global → core area → local population.

The scientific question for this session: we have a strong negative association between invasive species proportion and bull trout population growth rates (bINV_M ≈ −0.5 in full model, z ≈ −3 to −4). However, we are concerned this reflects correlation rather than causation, and we want to strengthen the causal inference or at minimum characterize the direction of influence.

The two competing hypotheses are: - Displacement: invasives cause bull trout decline (INV → ↓BLT). Temporal signature: invasive increase precedes bull trout decline (INV leads BLT). - Replacement: bull trout decline enables invasive expansion (↓BLT → INV). Temporal signature: bull trout decline precedes invasive increase (BLT leads INV). This could occur if climate stress reduces bull trout before invasives arrive, and invasives simply fill the vacated niche.

Both processes can occur simultaneously, and in practice both are likely.

Current Model Structure (relevant to INV)

C++ file: Code/MPVA_TMB_BUILDING_1_23.cpp Run script: Code/Run_MPVA_Model1_23.R Functions: Code/MPVA_Functions1_14.R Data prep: Code/PrepareData_MPVA1_8.R

Process equation (simplified)

logN(p,t) = logN(p,t-1) + R(p,t) + epsN(p,t-1)

R(p,t) = b0r_pop[p]
        + bFlow_pop[p]  * Flow(p, t-1)     # lagged 6 yr in data matrix
        + bTemp_pop[p]  * Temp(p, t-1)     # lagged 6 yr in data matrix
        + bTemp2        * Temp(p, t-1)^2   # global quadratic
        + bINV_pop[p]   * INV(p, t-1)      # ONE built-in t-1 lag (C++ index)
        + bLKT_M        * LKT(p, t-1)      # lake trout only, global scalar
        - exp(log_b0phi_pop[p]) * N(p,t-1) / extent[p] / 1000   # density dep.

INV data details

  • Metric: logit(invasive proportion) z-scored, where invasive proportion = (lake trout
    • brown trout CPUE) / (all salmonid CPUE) at each patch, each year
  • Data matrix: INV is nPop × nYears; inv_years_needed = DatYears (no additional lag shift applied in data prep — the t-1 lag is entirely within the C++ loop)
  • Hierarchy: bINV_pop[p] ~ N(bINV_M, sigma_bINV) — population-level random slopes drawn from a global mean
  • Current result: bINV_M ≈ −0.5 (negative growth effect of invasives)

Key parameters for INV

PARAMETER(bINV_M);               // global mean invasive effect
PARAMETER_VECTOR(bINV_pop);      // pop-level random slopes [nPop]
PARAMETER(log_sigma_bINV);       // SD of pop-level variation

Data Availability for Causal Analysis

The INV data is annual (or interpolated to annual) for most populations over roughly 1980–2020. The primary invasive species are lake trout (Salvelinus namaycush) in lake-connected systems and brown trout (Salmo trutta) in stream systems.

Bull trout redd counts (ReddCounts) are the primary abundance index, supplemented by CPUE electrofishing data (CPUEobs) at some populations.

Important: Flow (bFlow_pop) is already lagged 6 years to represent cohort effects through the juvenile stage (spawning flow → juvenile survival → adult recruitment). Temperature similarly. INV currently has only the 1-step (1-year) lag built into the C++ loop.

What We Want to Investigate

1. Optimal lag for INV effect

The 1-year lag (t-1) in the C++ loop may be too short biologically. If invasives reduce bull trout via competition with juveniles, the signal would propagate through the 3–5 year juvenile period before appearing in redd counts. This mirrors the 6-year lag used for flow.

Question: What lag length (0–8 years) produces the best AICc for the INV effect? This would use the same ModelSelection_Lags framework already in the codebase.

2. Cross-lag (Granger-style) analysis

To distinguish displacement from replacement, we can test asymmetric cross-lagged effects: - Does INV(t-k) predict BLT residuals (BLT growth unexplained by Ricker model)? → displacement - Does BLT(t-k) predict INV(t) change? → replacement/competitive release

This is most naturally done as a supplementary analysis on model residuals, not within the main TMB model.

3. INV × Temperature interaction

Under the climate facilitation hypothesis, invasives are most damaging when temperatures are already stressful for bull trout (near the upper thermal limit). An INV × Temp interaction term would test whether invasive effects are stronger in warm years/populations.

This requires adding an interaction parameter to the C++ model.

4. Multi-lag INV in the model

Rather than a single lag, we could include INV at multiple lags simultaneously (e.g., t-1 and t-4) to capture both immediate competition effects and longer-term recruitment effects.

Suggested Starting Points (ranked by effort vs insight)

Priority Analysis Effort Insight
1 Lag selection for INV (0–7 yr) using existing lag framework Low High — tells us biological timescale
2 Cross-lag regression on model residuals (Granger-style) Medium High — direction of causation
3 INV × Temp interaction in C++ model Medium Medium — climate facilitation test
4 Multi-lag INV (t-1 + t-k) High Medium — may over-parameterise

Recommended first step: Adapt ModelSelection_Lags_1_2.R to test INV lags 0–7 while holding Flow (lag 6) and Temp (lag 6) fixed. This will tell us whether the 1-year lag is biologically appropriate and whether a longer lag improves fit.

Second step: Cross-lag residual analysis — extract log-lambda residuals from the best model, regress them on past INV, and separately regress INV changes on past log-lambda. Compare R² / AIC of INV→BLT vs BLT→INV regressions.
